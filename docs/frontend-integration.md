# Integrating ShelbyBlobStore with the Frontend

This document explains how to integrate the ShelbyBlobStore Move contract with the frontend application.

## Overview

The frontend now displays transaction hashes after successful uploads. The transaction hash is obtained from the wallet's `signAndSubmitTransaction` function response.

## Component Updates

The `ShelbyUploader.tsx` component has been updated to:

1. Store the transaction hash in component state
2. Display the transaction hash after a successful upload
3. Provide a link to view the transaction on the Aptos Explorer

## Implementation Details

### Transaction Hash Storage

The transaction hash is stored in the `txHash` state variable:

```typescript
const [txHash, setTxHash] = useState<string | null>(null);
```

### Displaying Transaction Hash

After a successful transaction, the component displays:

1. The transaction hash in a truncated format
2. A link to view the transaction on Aptos Explorer

### Code Example

```typescript
{txHash && (
  <div className="mt-2 text-xs">
    <span className="font-medium">Transaction Hash:</span>
    <div className="font-mono text-emerald-800 truncate">{txHash}</div>
    <a 
      href={`https://explorer.aptos.dev/txn/${txHash}`} 
      target="_blank" 
      rel="noopener noreferrer"
      className="text-emerald-700 hover:text-emerald-900 underline mt-1 inline-block"
    >
      View on Aptos Explorer
    </a>
  </div>
)}
```

## Future Integration with Smart Contract

To fully integrate with the Move smart contract:

1. Deploy the contract to the Aptos network
2. Update the upload process to call the `upload_blob` function
3. Pass the asset ID generated by the frontend to the smart contract
4. Listen for `BlobUploaded` events to confirm on-chain storage

## Testing

To test the current implementation:

1. Connect your Petra wallet
2. Select an image file to upload
3. Complete the transaction
4. Verify that the transaction hash is displayed
5. Click the "View on Aptos Explorer" link to verify the transaction
